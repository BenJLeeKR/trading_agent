# Priority 5-1 Runtime Wiring: bootstrap → orchestrator provider agent 연결

## 목적

`bootstrap.py`에서 `AppSettings`의 provider 설정을 읽어 `OpenAICompatibleClient`와 `EventInterpretationAgent`(real)를 생성하고, `DecisionOrchestratorService`에 주입한다. Provider 설정이 없으면 기존 `StubEventInterpretationAgent` fallback을 유지한다.

## 0. 사용자 추가 조건 (승인됨)

1. **확장 가능한 helper 설계**: 지금은 `EventInterpretationAgent`만 real이지만, 향후 `AIRiskAgent`/`FinalDecisionComposerAgent` real 구현 시 helper 구조가 확장을 막지 않도록 작성
2. **3중 검증**: `provider_api_key`뿐 아니라 `provider_base_url`과 `provider_model_id`까지 모두 설정되어 있어야 real agent 생성. 하나라도 빠지면 `logging.warning` 후 stub fallback

## 2. 상세 변경 내용

### 2.1 `bootstrap.py` — `_build_provider_agent()` helper

**확장 가능한 설계**: 인자로 `provider_agent_type`을 받지 않고, `EventInterpretationAgent`만 직접 반환. 향후 Agent 2/3 real 구현 시 별도 `_build_ai_risk_agent()` / `_build_final_decision_agent()` helper 추가 가능 (naming 일관성 유지).

```python
def _build_provider_agent(settings: AppSettings) -> EventInterpretationAgent | None:
    """Build a real EventInterpretationAgent if provider settings are complete.

    Validates that **all three** of ``provider_api_key``, ``provider_base_url``,
    and ``provider_model_id`` are configured.  If any is missing/empty, logs a
    warning and returns ``None`` so the orchestrator falls back to
    ``StubEventInterpretationAgent``.

    Returns ``None`` when provider settings are incomplete — the caller
    (orchestrator) silently falls back to ``StubEventInterpretationAgent``.
    """
    if not settings.provider_api_key:
        logger.info("Provider API key not configured — using stub EventInterpretationAgent")
        return None
    if not settings.provider_base_url:
        logger.warning("provider_base_url is empty — using stub EventInterpretationAgent")
        return None
    if not settings.provider_model_id:
        logger.warning("provider_model_id is empty — using stub EventInterpretationAgent")
        return None

    client = OpenAICompatibleClient(
        api_key=settings.provider_api_key,
        base_url=settings.provider_base_url,
        timeout_seconds=settings.provider_timeout_seconds,
    )
    return EventInterpretationAgent(
        provider_client=client,
        model_id=settings.provider_model_id,
    )
```

**설계 결정**:
- `provider_api_key`가 비어있으면 `None` 반환 → orchestrator의 기본값(`StubEventInterpretationAgent()`)이 사용됨
- `provider_api_key`가 있으면 real `OpenAICompatibleClient` + `EventInterpretationAgent` 생성
- `OpenAICompatibleClient`의 `close()`는 orchestrator shutdown 시 별도로 호출 필요 (아래 2.3 참조)

### 2.3 Cleanup: `shutdown_postgres_runtime()`에서 provider client close

`EventInterpretationAgent`가 내부에 `OpenAICompatibleClient`를 들고 있으므로, runtime 종료 시 `close()` 호출 필요.

**Private attribute 최소화 전략**: `hasattr()` + `callable()` 이중 검사로 안전하게 접근.

```python
async def _close_provider_agent(agent: object | None) -> None:
    """Safely close the provider agent's underlying HTTP client.

    Uses ``hasattr`` + ``callable`` checks to avoid direct private
    attribute access assumptions.
    """
    if agent is None:
        return
    provider = getattr(agent, "_provider", None)
    if provider is not None and callable(getattr(provider, "close", None)):
        await provider.close()


async def shutdown_postgres_runtime(runtime: dict[str, Any]) -> None:
    agent = runtime.get("event_interpretation_agent")
    await _close_provider_agent(agent)
    await close_pool()
```

**참고**: 추후 `AsyncCloseable` protocol을 도입하면 `_close_provider_agent()`를 정식 메서드로 대체 가능.
