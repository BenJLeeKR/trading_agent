## 1. 화면 목적

- 운영자가 **선택한 종목 1개의 실시간 현재가·호가·거래 정보**를 HTS 수준의 상세도로
  한눈에 확인한다. 여러 종목을 동시에 나열해 비교하는 대시보드가 아니라, "지금 이
  종목이 어떻게 움직이고 있는가"를 깊이 있게 보여주는 조회 화면이다.
- 이 화면은 **조회 전용(read-only)** 운영 도구다. 주문 제출, 자동매매 판단, universe
  편입, signal 계산과는 어떤 UI 경로로도 연결하지 않는다
  (`11_...md` §2.2 비범위, §7 안전장치와 동일한 원칙).
- 1차 목표는 "예쁜 대시보드"가 아니라, **운영자가 지금 이 화면의 숫자를 얼마나 믿을 수
  있는지(연결 상태, 마지막 수신 시각, 41건 한도 여유)를 항상 함께 보여주는 것**이다.

## 2. 메뉴 위치

- 위치: **"기본 운영"** 섹션 (`Layout.tsx`의 `navSections` 배열 중 `title: "기본 운영"` 그룹,
  현재 항목: 주문 / 체결내역 / 계좌 / 의사결정 / 에이전트 실행)
- 메뉴명: **"실시간 현재가"**
- 아이콘(제안): `lucide-react`의 `LineChart` (기존 메뉴 아이콘 `FileText`/`ListOrdered`/
  `Wallet`/`Brain`/`Zap`과 계열이 겹치지 않는 시계열/시세 전용 아이콘)
- 삽입 위치: "기본 운영" 그룹의 마지막 항목("에이전트 실행") 다음.

## 3. 라우트

```
/operations/realtime-quotes
```

> 참고: 경로 접두사가 `/operations/`이지만 메뉴상 위치는 "운영 대시보드" 섹션이 아니라
> "기본 운영" 섹션이다. 기존 코드에도 `기본 운영` 섹션 항목이 `/operations/` 접두사를
> 쓰지 않는 평평한 경로(`/orders`, `/fills` 등)인 선례가 있으므로, 이 화면만 `/operations/`
> 접두사를 쓰는 것이 어색할 수 있다 — **최종 경로 문자열은 구현 착수 시 재확인**하되,
> 사용자가 지정한 `/operations/realtime-quotes`를 이 문서의 기준값으로 고정한다.
- 선택된 종목은 쿼리 파라미터로 유지한다: `/operations/realtime-quotes?symbol=138040`
  (새로고침/링크 공유 시 같은 종목이 다시 열리도록).

### 3.1 다른 화면에서의 진입 (딥링크)

**결론: 가능하며, 기존 코드에 이미 동일한 패턴의 선례가 있다.**

- `AccountsView.tsx:340`은 포지션 행에서 "관련 주문 보기 →" 버튼 클릭 시
  `navigate(\`/orders?symbol=${encodeURIComponent(r.symbol)}\`)`로 종목코드를 쿼리
  파라미터로 넘긴다.
- `OrdersView.tsx:60-66`은 마운트 시 `location.search`에서 `symbol` 파라미터를 읽어
  검색창을 자동으로 채운다(`useEffect` + `URLSearchParams`).
- 이 화면도 **동일한 패턴을 그대로 재사용**한다: `/operations/realtime-quotes?symbol=138040`으로
  진입하면, 마운트 시 해당 종목을 자동으로 구독 추가하고 메인 뷰(C/D/E)에 즉시 표시한다
  (단순 검색창 채움이 아니라 "그 종목으로 바로 조회"까지 수행한다는 점이 `OrdersView`의
  기존 동작보다 한 단계 더 나아간 부분).

**단, 현재 확인 결과 종목을 클릭 가능하게 만드는 작업은 아직 어디에도 없다.**
`OrdersView.tsx:108-109`와 `FillHistoryView.tsx:84`의 종목(symbol) 컬럼은 현재
**클릭 불가능한 일반 텍스트**로 렌더링된다(`<span>{r.symbol}</span>` / `row.symbol`
그대로 출력). 따라서 "체결내역/주문 화면에서 종목 클릭 → 이 화면으로 이동"을 완성하려면,
**이 신규 화면 자체의 구현과는 별개로 다음 두 화면에 작은 추가 작업이 필요하다**:

| 화면 | 필요한 변경 | 참고 선례 |
|---|---|---|
| `OrdersView.tsx` | 종목(symbol) 컬럼 `render`를 클릭 가능한 링크/버튼으로 변경, `navigate(\`/operations/realtime-quotes?symbol=${encodeURIComponent(r.symbol)}\`)` 호출 | `AccountsView.tsx:340`의 "관련 주문 보기 →" 버튼과 동일한 방식 |
| `FillHistoryView.tsx` | 동일하게 종목 컬럼을 클릭 가능하게 변경 | 위와 동일 |

이 두 변경은 **이 신규 화면의 컴포넌트 작업이 아니라 기존 화면에 대한 소규모 추가 작업**이므로,
`11_kis_realtime_quote_operations_screen.md`의 단계별 계획(§8) 중 **Phase 3(Admin UI polling
화면) 범위에 명시적으로 포함**시키는 것을 권장한다 (이 문서 자체는 UI 레이아웃 문서이므로
그 문서의 실제 수정은 여기서 하지 않는다).

## 5. 주요 UI 영역

| # | 영역 | 역할 |
|---|---|---|
| A | 연결 상태/환경 표시 | 이 화면 전용 KIS WS 세션의 연결 여부, 마지막 수신 시각, `LIVE` 환경 고정 배지, 41건 구독 한도 |
| B | 종목 전환 바 | 종목코드 검색/입력 + 현재 구독 중인 종목들을 탭/chip으로 빠르게 전환 |
| C | 종목 헤더 + 현재가 바 | 선택된 종목의 시장 구분(KOSPI/KOSDAQ) 배지, 종목명, 현재가/등락/등락률, 전일종가 |
| D | 10단계 호가창(ladder) | 선택된 종목의 매도 10단계 + 매수 10단계 호가와 잔량, 잔량합계, 시간외 잔량 |
| E | 종목 상세정보 패널 | 시간구분, 거래정지 여부, VI발동기준가, 상한가/고가/시가/저가/하한가/기준가, 누적·전일 거래량/거래대금 |
| F | 오류/재연결 상태 | WS 연결 문제, REST fallback 사용 여부를 명시적으로 노출 |
| G | 수신 상태 표시 | 선택된 종목 기준 마지막 수신 시각, 데이터 출처(WS 실시간/REST 보정) |

### A. 연결 상태/환경 표시

### B. 종목 전환 바

| 필드 | 설명 |
|---|---|
| 종목코드 드롭다운/검색 | 6자리 코드 직접 입력 또는 이름 검색 → 선택 시 해당 종목이 메인 뷰(C/D/E)로 전환 |

### C. 종목 헤더 + 현재가 바

| 필드 | 설명 | 데이터 출처 |
|---|---|---|
| 시장 구분 배지 | `KOSPI` / `KOSDAQ` (KRX 전용이므로 이 두 값만) | instrument master 또는 REST 조회 응답 |
| 종목명 | 한글 종목명 | instrument master |
| 현재가 | 체결가, 상승/하락에 따라 빨강/파랑 | `H0STCNT0.STCK_PRPR` |
| 전일대비 + 등락률 | 화살표(▲/▼/-) + 값 + 괄호 안 퍼센트 | `PRDY_VRSS` + `PRDY_VRSS_SIGN` + `PRDY_CTRT` |
| 전일종가(기준가) | 참고 이미지 우측 상단의 회색 숫자 | 아래 "정적 참조값" 참고 |

> REST 현재가 조회(`FHKST01010100`, 기존 코드에 이미 구현된 `KISRestClient.get_quote()`)의

## 7. Loading / Empty / Error / Degraded 상태

| 상태 | 트리거 조건 | 화면 동작 |
|---|---|---|
| **Loading** | 최초 진입 또는 종목 전환 직후, snapshot 응답 대기 중 | 호가창/상세정보 영역에 스켈레톤 또는 스피너, 연결 상태는 "확인 중" 표시 |
| **Empty** | 구독 중인 종목이 0개(최초 진입, 아직 아무 종목도 검색하지 않음) | 메인 뷰(C/D/E) 대신 "종목을 검색해 조회를 시작하세요" 안내 + 검색창 강조. 종목 전환 바는 빈 상태 유지 |
| **Error** | bootstrap/subscribe/snapshot API 호출 실패(4xx/5xx, adapter 미설정 503 포함) | 화면 상단에 에러 배너, 마지막으로 성공한 데이터가 있으면 그 값을 유지하며 "최신 데이터 아님" 표시 |
| **Degraded** | WS 연결 끊김 + REST fallback 사용 중, 또는 재연결 시도 중 | F 영역 배너 노출, 호가창/현재가 값에 "지연/stale" 표시, 화면 전체를 막지 않고 마지막 값은 계속 보여줌 |

원칙: **연결이 불안정해도 화면 전체를 비우지 않는다.** 마지막으로 확인된 값 + "언제
확인된 값인지"를 항상 함께 보여주는 것이 이 화면의 핵심 신뢰성 요구사항이다
(`11_...md` §4.6 disconnect/reconnect 정책과 일치).

## 8. Desktop / Mobile Responsive 기준

- **Desktop (≥1024px, 기준 레이아웃)**: §4.1 와이어프레임. 좌측 사이드바 + 본문 영역에서
  10단계 호가창(D)과 종목 상세정보 패널(E)을 좌우 2-column으로 배치(호가창 넓게, 상세
  정보 우측 고정폭 — 참고 이미지와 동일 비율).
- **Tablet (768px~1023px)**: 종목 상세정보 패널(E)은 호가창(D) 아래로 이동
  (2-column → 1-column stack). 종목 전환 바는 그대로 상단 고정.
- **Mobile (<768px, §4.2 와이어프레임)**: 화면 폭 제약상 10단계 전체를 항상 펼쳐두지
  않는다 — **기본은 매도/매수 각 5단계만 표시하고 "더보기"로 10단계 전체 확장**한다.
  종목 상세정보 패널(E)은 접이식(accordion) 섹션으로 전환. 종목 전환 탭은 가로 스크롤
  가능한 한 줄로 유지.
- 공통 원칙: 연결 상태 배지와 `LIVE` 환경 배지, 현재가/등락 정보(C)는 모든 breakpoint에서
  항상 최상단에 고정 노출한다(환경 혼동 방지 원칙, 화면 크기와 무관하게 유지).

## 10. Phase 1에서 제외할 과한 기능

다음은 이 화면의 1차 목표를 넘어서는 기능이며, Phase 1 레이아웃/구현 범위에서
명시적으로 제외한다.

- **차트(캔들/라인) 시각화**: 시계열 차트는 별도의 시각화 라이브러리 도입과 성능
  고려가 필요한 큰 작업이며, 이 화면은 "지금 값"을 보여주는 것이 목적이지 추세
  분석 도구가 아니다.
- **다중 종목 동시 화면 비교(2개 이상 종목을 나란히 배치)**: 이번 재설계로 화면은
  단일 종목 상세 조회 중심이 되었다 — 여러 종목을 한 화면에서 나란히 비교하는
  뷰(v1의 다중 종목 테이블)는 Phase 1에서 제외한다. 여러 종목 확인은 종목 전환
  바(B)로 빠르게 전환하는 방식으로 대체한다.
- **종목 즐겨찾기/저장된 워치리스트 영속화**: 새로고침/재접속 시 구독 목록을 서버에
  영속 저장하는 기능은 Phase 1 범위 밖(`11_...md` §4.5, 메모리 전용 원칙과 일관).
  Phase 1은 세션 동안만 구독 목록(종목 전환 바의 탭 목록)을 유지한다.
- **알림/알람(가격 도달 알림, 급등락 알림)**: 알림 인프라는 `11_...md` §8 Phase 5
  "운영 관측/alert 연동"의 범위이며, 이 화면 레이아웃과는 별개 작업이다.
- **예상체결가/예상체결량 상세 표시(동시호가 구간)**: 178번 문서의 `ANTC_CNPR` 등
  예상체결 필드는 동시호가 등 특정 조건에서만 발생하는 값이라 Phase 1 상세정보
  패널(E)에는 포함하지 않는다. 필요성이 확인되면 후속 단계에서 추가한다.
- **모바일 전용 별도 네이티브 UX(스와이프 삭제 등)**: §8의 반응형 웹 레이아웃까지만
  다루고, 모바일 전용 제스처 인터랙션은 범위 밖이다.
- **주문 관련 CTA(예: "이 호가에 매수" 버튼, 호가 클릭 시 주문창 연결)**: 이 화면에서
  주문 화면으로의 어떤 단축 진입 동선도 만들지 않는다 — read-only 원칙(`11_...md` §7)을
  UI 레벨에서도 엄격히 지킨다. 참고 이미지 스타일의 HTS 화면은 보통 호가 클릭 시
  주문창에 값이 채워지는 기능이 있지만, 이 화면은 그 기능을 의도적으로 제공하지 않는다.
