## 1. 화면 목적

- 운영자가 선택한 종목의 **실시간 현재가, 등락, 체결 시각, 수신 상태, 연결 상태**를
  한 화면에서 빠르게 확인한다.
- 이 화면은 **조회 전용(read-only)** 운영 도구다. 주문 제출, 자동매매 판단, universe
  편입, signal 계산과는 어떤 UI 경로로도 연결하지 않는다
  (`11_...md` §2.2 비범위, §7 안전장치와 동일한 원칙).
- 1차 목표는 "예쁜 대시보드"가 아니라, **운영자가 지금 이 화면의 숫자를 얼마나 믿을 수
  있는지(연결 상태, 마지막 수신 시각, 41건 한도 여유)를 항상 함께 보여주는 것**이다.

## 2. 메뉴 위치

- 위치: **"기본 운영"** 섹션 (`Layout.tsx`의 `navSections` 배열 중 `title: "기본 운영"` 그룹,
  현재 항목: 주문 / 체결내역 / 계좌 / 의사결정 / 에이전트 실행)
- 메뉴명: **"실시간 현재가"**
- 아이콘(제안): `lucide-react`의 `LineChart` (기존 메뉴 아이콘 `FileText`/`ListOrdered`/
  `Wallet`/`Brain`/`Zap`과 계열이 겹치지 않는 시계열/시세 전용 아이콘)
- 삽입 위치: "기본 운영" 그룹의 마지막 항목("에이전트 실행") 다음.

## 3. 라우트

```
/operations/realtime-quotes
```

> 참고: 경로 접두사가 `/operations/`이지만 메뉴상 위치는 "운영 대시보드" 섹션이 아니라
> "기본 운영" 섹션이다. 기존 코드에도 `기본 운영` 섹션 항목이 `/operations/` 접두사를
> 쓰지 않는 평평한 경로(`/orders`, `/fills` 등)인 선례가 있으므로, 이 화면만 `/operations/`
> 접두사를 쓰는 것이 어색할 수 있다 — **최종 경로 문자열은 구현 착수 시 재확인**하되,
> 사용자가 지정한 `/operations/realtime-quotes`를 이 문서의 기준값으로 고정한다.

## 5. 주요 UI 영역

| # | 영역 | 역할 |
|---|---|---|
| A | 연결 상태/환경 표시 | 이 화면 전용 KIS WS 세션의 연결 여부, 마지막 수신 시각, `LIVE` 환경 고정 배지 |
| B | 종목 검색/코드 입력 | 종목코드 직접 입력 또는 이름 검색으로 구독 추가 |
| C | 구독 중인 종목 목록 | 현재 구독 중인 전체 종목을 chip 형태로 나열, 개별 해제 가능 |
| D | 실시간 현재가 테이블 | 구독 종목 전체를 한 번에 비교하는 테이블 뷰 |
| E | 상세 quote panel | 테이블에서 선택한 1개 종목의 상세 시세(호가 포함) |
| F | 오류/재연결 상태 | WS 연결 문제, REST fallback 사용 여부를 명시적으로 노출 |
| G | 구독 한도/수신 지연 표시 | 41건 대비 사용량, 종목별 마지막 수신 후 경과 시간 |

## 6. 영역별 표시 필드

### A. 연결 상태/환경 표시

| 필드 | 설명 | 데이터 출처(예정) |
|---|---|---|
| 연결 상태 배지 | 연결됨 / 재연결 중 / 끊김 | `GET /realtime-quotes/bootstrap`(§5.1, 11_...md) |
| 환경 배지 | 항상 `LIVE` 고정, 회색이 아닌 명확한 색상으로 강조 | 클라이언트 고정값 |
| 마지막 수신 시각 | 이 화면 전체 기준 가장 최근 tick 수신 시각(상대 시간, 예: "3초 전") | snapshot 응답의 최신 timestamp |
| 사용 계좌 안내 | "실시간시세 전용 계좌(트레이딩 계좌와 다름)" 툴팁 | 정적 텍스트 (11_...md §6.3) |

## 7. Loading / Empty / Error / Degraded 상태

| 상태 | 트리거 조건 | 화면 동작 |
|---|---|---|
| **Loading** | 최초 진입, `GET /realtime-quotes/bootstrap` 응답 대기 중 | 테이블/패널 영역에 로딩 스피너, 연결 상태는 "확인 중" 표시 |
| **Empty** | 구독 중인 종목이 0개 | 테이블 대신 "구독 중인 종목이 없습니다. 종목을 검색해 추가하세요." 안내 + 검색창 강조 |
| **Error** | bootstrap/subscribe/snapshot API 호출 실패(4xx/5xx, adapter 미설정 503 포함) | 화면 상단에 에러 배너, 마지막으로 성공한 데이터가 있으면 그 값을 유지하며 "최신 데이터 아님" 표시 |
| **Degraded** | WS 연결 끊김 + REST fallback 사용 중, 또는 재연결 시도 중 | F 영역 배너 노출, 테이블 행에 "지연/stale" 표시, 전체 화면을 막지 않고 마지막 값은 계속 보여줌 |

원칙: **연결이 불안정해도 화면 전체를 비우지 않는다.** 마지막으로 확인된 값 + "언제
확인된 값인지"를 항상 함께 보여주는 것이 이 화면의 핵심 신뢰성 요구사항이다
(`11_...md` §4.6 disconnect/reconnect 정책과 일치).

## 8. Desktop / Mobile Responsive 기준

- **Desktop (≥1024px, 기준 레이아웃)**: §4.1 와이어프레임. 좌측 사이드바 + 본문 영역에서
  연결상태/구독한도가 상단 2-column, 테이블과 상세 패널이 좌우 분할(테이블 넓게, 상세
  패널 우측 고정폭) 또는 테이블 아래 상세 패널 배치 중 구현 시 기존 `OrderTrackingView`/
  `ReconciliationView` 레이아웃 컨벤션을 따른다.
- **Tablet (768px~1023px)**: 상세 quote panel은 테이블 아래로 이동(2-column → 1-column
  stack). 구독 한도/연결 상태 카드는 1-column으로 세로 배치.
- **Mobile (<768px, §4.2 와이어프레임)**: 모든 카드가 1-column 세로 스택. 테이블은
  가로 스크롤 허용(컬럼 축소 없이 스크롤) 또는 핵심 컬럼(종목/현재가/등락률)만 표시하고
  나머지는 행 탭 시 모달/바텀시트로 노출. 종목 chip 목록은 가로 스크롤 가능한 한 줄.
- 공통 원칙: 연결 상태 배지와 `LIVE` 환경 배지는 모든 breakpoint에서 항상 최상단에
  고정 노출한다(6.3의 환경 혼동 방지 원칙, 화면 크기와 무관하게 유지).

## 10. Phase 1에서 제외할 과한 기능

다음은 이 화면의 1차 목표("운영자가 현재가를 빠르게 확인")를 넘어서는 기능이며,
Phase 1 레이아웃/구현 범위에서 명시적으로 제외한다.

- **호가창 전체(10단계 매도/매수 호가) 표시**: 상세 quote panel(E)은 Phase 1에서
  1호가만 보여준다. 10단계 호가창 UI는 트레이딩 판단용 정보에 가까워 이 화면의
  "운영 편의 조회" 목적을 넘어선다.
- **차트(캔들/라인) 시각화**: 시계열 차트는 별도의 시각화 라이브러리 도입과 성능
  고려가 필요한 큰 작업이며, 이 화면은 "지금 값"을 보여주는 것이 목적이지 추세
  분석 도구가 아니다.
- **종목 즐겨찾기/저장된 워치리스트 영속화**: 새로고침/재접속 시 구독 목록을 서버에
  영속 저장하는 기능은 Phase 1 범위 밖(`11_...md` §4.5, 메모리 전용 원칙과 일관).
  Phase 1은 세션 동안만 구독 목록을 유지한다.
- **알림/알람(가격 도달 알림, 급등락 알림)**: 알림 인프라는 `11_...md` §8 Phase 5
  "운영 관측/alert 연동"의 범위이며, 이 화면 레이아웃과는 별개 작업이다.
- **다중 종목 비교 뷰(정렬/필터 고도화, 등락률 기준 정렬 등)**: 테이블 기본 정렬/표시
  이상의 고급 필터링(업종별, 시가총액별 등)은 Phase 1에서 제외한다.
- **모바일 전용 별도 네이티브 UX(스와이프 삭제 등)**: §8의 반응형 웹 레이아웃까지만
  다루고, 모바일 전용 제스처 인터랙션은 범위 밖이다.
- **주문 관련 CTA(예: "이 가격에 매수" 버튼)**: 이 화면에서 주문 화면으로의 어떤
  단축 진입 동선도 만들지 않는다 — read-only 원칙(`11_...md` §7)을 UI 레벨에서도
  엄격히 지킨다.
