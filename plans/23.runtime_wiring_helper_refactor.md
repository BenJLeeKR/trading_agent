# Runtime Wiring Helper 리팩터 계획

## 문제점

1. `_build_orchestrator()` helper가 `bootstrap.py`에 존재하지만, 세 runtime factory(`build_default_runtime`, `build_postgres_runtime`, `postgres_runtime`) 모두 여전히 `DecisionOrchestratorService(...)`를 직접 생성하고 있음
2. Postgres runtime 경로(`build_postgres_runtime`, `postgres_runtime`)에 대한 wiring 테스트가 없음

## 변경 사항

### A. `bootstrap.py` — `_build_orchestrator()` 실제 사용

#### A.1 `_build_orchestrator()`에 선택적 `event_interpretation_agent` 파라미터 추가

```python
def _build_orchestrator(
    repos: RepositoryContainer,
    settings: AppSettings,
    event_interpretation_agent: EventInterpretationAgent | None = None,
) -> DecisionOrchestratorService:
    if event_interpretation_agent is None:
        event_interpretation_agent = _build_provider_agent(settings)
    return DecisionOrchestratorService(
        repos=repos,
        event_interpretation_agent=event_interpretation_agent,
    )
```

- 파라미터가 제공되지 않으면 내부적으로 `_build_provider_agent()` 호출 (기존 동작 유지)
- 파라미터가 제공되면 그대로 전달 (runtime dict용 agent와 동일 인스턴스 보장)

#### A.2 세 runtime factory에서 `_build_orchestrator()` 사용

각 factory에서:

```python
# 변경 전 (lines 159-163, 223-227, 293-297)
event_interpretation_agent = _build_provider_agent(settings)
orchestrator = DecisionOrchestratorService(
    repos=repositories,
    event_interpretation_agent=event_interpretation_agent,
)

# 변경 후
event_interpretation_agent = _build_provider_agent(settings)
orchestrator = _build_orchestrator(repositories, settings, event_interpretation_agent)
```

**영향**: `_build_provider_agent()`는 한 번만 호출되고, `_build_orchestrator()`가 실제로 사용됨. runtime dict의 `"event_interpretation_agent"` 키는 계속 유지됨.

### B. `test_bootstrap.py` — Postgres runtime 경로 테스트 추가

#### B.1 `TestBuildPostgresRuntime` — 5 tests

`build_postgres_runtime()` 검증. DB 의존성을 피하기 위해 `create_pool`, `ensure_schema`, `build_postgres_repositories` mocking.

| 테스트 | 검증 내용 |
|--------|----------|
| `test_contains_orchestrator` | runtime dict에 `orchestrator` 키, `DecisionOrchestratorService` 인스턴스 |
| `test_contains_event_interpretation_agent_key` | runtime dict에 `event_interpretation_agent` 키 |
| `test_uses_stub_when_no_api_key` | `DEEPSEEK_API_KEY` 없으면 agent는 `None` |
| `test_uses_real_agent_when_api_key_set` | 설정 있으면 `EventInterpretationAgent` 인스턴스 |
| `test_runtime_shape_consistent` | 7개 expected 키 모두 존재 (`db_config` 포함) |

#### B.2 `TestPostgresRuntimeContext` — 4 tests

`postgres_runtime()` context manager 검증. `transaction()` context manager도 mocking.

| 테스트 | 검증 내용 |
|--------|----------|
| `test_contains_orchestrator` | context 내부에서 `orchestrator` 키 존재 |
| `test_contains_event_interpretation_agent_key` | context 내부에서 `event_interpretation_agent` 키 존재 |
| `test_runtime_shape_consistent` | 7개 expected 키 모두 존재 |
| `test_shutdown_called_on_exit` | context exit 시 `shutdown_postgres_runtime()` 호출됨 |
