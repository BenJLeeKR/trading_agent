# 개발 실행 인프라 구성 계획 v2

## 목표

PostgreSQL 저장소 구현을 바로 시작할 수 있는 **최소 개발 실행 인프라**를 구성한다.
운영 인프라(Redis, observability, CI/CD, K8s, Terraform)는 이번 범위에서 제외한다.

## 현재 상태 vs 목표 상태

| 항목 | 현재 | 목표 |
|------|------|------|
| 프로젝트 정의 | 없음 | `pyproject.toml` |
| 의존성 | `.venv`는 있으나 정의 없음 | pyproject.toml에 선언 |
| 환경변수 | 코드에 하드코딩된 기본값 | `.env.example` 템플릿 |
| DB 연결 계층 | `connection.py` / `transaction.py` / `migrations/run.py` | **신규 작성** 수준으로 정비 |
| Git | 없음 | `.gitignore` |
| DB 실행 | 없음 | `docker-compose.yml`에 PostgreSQL 포함 |
| 앱 컨테이너 | 없음 | `Dockerfile` (부팅 + DB 연결 + migration 실행 가능) |
| 태스크 실행 | 없음 | `Makefile` |
| 개발 가이드 | 없음 | `README.md` |
| 테스트 실행 | `pytest.ini` 있음 (`asyncio_mode=auto`) | 유지, 추가 보강 불필요 |

## 구현할 파일 (8개 신규 + 1개 재작성)

### 1. `pyproject.toml` (신규)

**도구**: `setuptools` — Poetry는 학습 비용 대비 이점이 없고, `pip install -e ".[dev]"`로 충분.

```toml
[build-system]
requires = ["setuptools>=68.0"]
build-backend = "setuptools.backends._legacy:_Backend"

[project]
name = "agent-trading"
version = "0.1.0"
description = "AI Multi-Agent Trading System"
requires-python = ">=3.11"
dependencies = [
    "asyncpg>=0.29",
    "python-dotenv>=1.0",
]
[project.optional-dependencies]
dev = [
    "pytest>=8.0",
    "pytest-asyncio>=0.24",
]
[project.scripts]
agent-trading = "agent_trading.main:main"
[tool.setuptools.packages.find]
where = ["src"]
include = ["agent_trading*"]
```

### 3. `connection.py` (재작성)

**변경 사항**:
- `DatabaseConfig`: `DATABASE_*` env var 우선, `DB_*` fallback, `schema` 필드 추가
- `create_pool()`: 안전한 pool 초기화 플래그 사용 (`_pool._closed` private 접근 제거)
- `create_pool()`: schema-aware connection init 옵션 (`SET search_path TO trading`)
- `health_check()`: 간단한 DB 연결 확인 함수 추가

**env var 해석 순서**:
```
DATABASE_HOST → DB_HOST → "localhost"
DATABASE_PORT → DB_PORT → "5432"
DATABASE_NAME → DB_NAME → "trading"
DATABASE_USER → DB_USER → "trading"
DATABASE_PASSWORD → DB_PASSWORD → "trading"
DATABASE_SCHEMA → "trading"
```

### 4. `transaction.py` (재작성)

### 7. `Dockerfile` (신규)

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY pyproject.toml .
RUN pip install --no-cache-dir -e ".[dev]"
COPY src/ src/
COPY db/ db/
ENV PYTHONPATH=/app/src
ENV PYTHONUNBUFFERED=1
CMD ["agent-trading"]
```

- `python:3.12-slim` — 경량, 의존성 충분
- `[dev]` 설치로 pytest 포함
- `PYTHONPATH`로 `src/` 레이아웃 지원

### 8. `docker-compose.yml` (신규)

```yaml
services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: trading
      POSTGRES_USER: trading
      POSTGRES_PASSWORD: trading
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U trading"]
      interval: 3s
      timeout: 3s
      retries: 5

  app:
    build: .
    depends_on:
      db:
        condition: service_healthy
    environment:
      DATABASE_HOST: db
      DATABASE_PORT: 5432
      DATABASE_NAME: trading
      DATABASE_USER: trading
      DATABASE_PASSWORD: trading
      DATABASE_SCHEMA: trading
    volumes:
      - ./src:/app/src
      - ./db:/app/db
      - ./tests:/app/tests
    command: ["tail", "-f", "/dev/null"]

volumes:
  pgdata:
```

- **PostgreSQL 16 Alpine** + **App (Python 3.12 slim)**
- healthcheck로 DB 준비 완료 보장
- 볼륨 마운트로 소스코드 변경 실시간 반영
- app은 대기 모드 → `docker compose exec app ...` 로 명령 실행
- Redis, observability 등 불필요한 서비스 없음

### 9. `Makefile` (신규)

```makefile
.PHONY: install run migrate test lint docker-up docker-down \
        docker-migrate docker-test docker-shell

install:
	pip install -e ".[dev]"

run:
	python -m agent_trading.main

migrate:
	python -m agent_trading.db.migrations.run

test:
	python -m pytest tests/ -v

lint:
	python -m pip install ruff && python -m ruff check src/

docker-up:
	docker compose up -d

docker-down:
	docker compose down

docker-migrate:
	docker compose exec app python -m agent_trading.db.migrations.run

docker-test:
	docker compose exec app python -m pytest tests/ -v

docker-shell:
	docker compose exec app /bin/bash
```
