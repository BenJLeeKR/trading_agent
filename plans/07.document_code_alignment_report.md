## 4. 결정 사항

1. **문서를 목표 기준선으로 삼는다.** 현재 코드는 부분 구현 상태로, 문서가 우선한다.
2. **다음 마일스톤(Milestone 4)은 "Safe Order Path Persistence Gap Closure"**로 범위를 축소한다.
3. Milestone 4에는 **Safe Order Path와 Runtime Safety에 직접 연결되는 항목만** 포함한다.
4. Config 전체 스키마, TradeDecisionEntity 전체 확장, SubmitOrderRequest/Result 확장, BrokerAdapter 확장, KIS adapter 실제 구현은 이후 마일스톤으로 분리한다.

> 상세 계획은 [`milestone4_safe_order_path_persistence.md`](milestone4_safe_order_path_persistence.md) 참조.
