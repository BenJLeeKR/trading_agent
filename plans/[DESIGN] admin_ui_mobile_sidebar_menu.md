# Admin UI 모바일 사이드바 → 삼선 메뉴 전환 설계

> **상태**: ✅ 설계+구현 완료 (2026-07-09)
> **범위**: `admin_ui/src/components/Layout.tsx` 1개 파일. 새 라이브러리/의존성 없음.
> 백엔드/라우팅/데이터 변경 없음.

## 1. 배경

현재 `Layout.tsx`의 좌측 사이드바(`w-[220px]`)는 반응형 분기(`md:`/`lg:` 등breakpoint) 없이
항상 고정 표시된다. 모바일 화면에서는 사이드바가 콘텐츠 영역을 과도하게 차지해
운영 화면을 보기 어렵다. 일반적인 모바일 패턴(좁은 화면에서는 사이드바를 숨기고
삼선(hamburger) 아이콘으로 대체, 탭하면 메뉴가 펼쳐짐)으로 전환한다.

## 2. 디자인 결정

| 항목 | 결정 |
|---|---|
| Breakpoint | Tailwind `md`(768px) 기준. `md` 이상 = 기존 데스크탑 사이드바 그대로, `md` 미만 = 모바일 모드 |
| 모바일 진입점 | 헤더 좌측에 삼선 아이콘 버튼(`lucide-react`의 `Menu`/`X` 아이콘, 신규 의존성 불필요) 추가. `md:hidden`으로 데스크탑에서는 노출 안 됨 |
| 펼쳐지는 방식 | 좌측에서 슬라이드되는 off-canvas 드로어가 아니라, **헤더 바로 아래로 펼쳐지는 드롭다운** 형태(요청사항 그대로) — 애니메이션 복잡도가 낮고 구현이 단순함 |
| 배경 처리 | 메뉴 펼침 중 콘텐츠 위에 반투명 backdrop(`bg-black/30`)을 깔아 탭하면 닫히도록 함(포커스 유도 + 실수 탭 방지) |
| 닫힘 트리거 | (1) 메뉴 항목 클릭/네비게이션 발생 시 자동 닫힘, (2) backdrop 클릭, (3) 삼선 아이콘 재탭(열림→닫힘 토글), (4) `Escape` 키 |
| 하단 로그아웃 | 기존 데스크탑 사이드바와 동일하게 모바일 드롭다운 맨 아래에도 포함 |
| 활성 라우트 강조 | 기존 데스크탑 로직(`NavLink` + `isActive` 배경색) 그대로 재사용 — 모바일 드롭다운에도 동일하게 적용 |
| 상태 관리 | `Layout` 컴포넌트 내 `useState<boolean>` 하나(`mobileMenuOpen`)로 충분. 전역 상태/컨텍스트 불필요 |
| 라우트 변경 시 자동 닫힘 | `useEffect`로 `location.pathname` 변화를 감지해 열려있으면 닫음(뒤로가기/딥링크 이동 시에도 메뉴가 열린 채 남지 않도록) |

## 3. 구조 변경 요약

- 기존 `<aside>`(데스크탑 사이드바)는 `hidden md:flex`로 조건부 표시만 추가, 내부 마크업/로직은 그대로 재사용.
- `navSections` 렌더링 로직(`NavLink`, `isActive`, `disabled` 처리)은 별도 함수/컴포넌트로 추출해 데스크탑 `<aside>`와 모바일 드롭다운이 동일 코드를 공유하도록 함(중복 방지).
- 헤더(`<header>`) 좌측에 `md:hidden` 삼선 버튼 추가.
- 모바일 드롭다운은 헤더 바로 아래 `absolute` 포지션 전체 폭 패널로 렌더링, `mobileMenuOpen` 상태에 따라 조건부 렌더.

## 4. 테스트 영향

- `admin_ui/src/__tests__/layout.test.tsx`의 기존 "always-visible sidebar" 가정 테스트는 데스크탑 시나리오(기본 jsdom 뷰포트)에서는 그대로 통과해야 함 — 사이드바 자체 마크업/텍스트는 변경하지 않으므로 회귀 없음.
- 신규 테스트: 삼선 버튼 클릭 시 모바일 드롭다운 노출/닫힘, 메뉴 항목 클릭 시 자동 닫힘 동작 검증 추가.

## 5. 비채택안

- **좌측 슬라이드 off-canvas 드로어**: 사용자가 명시적으로 "아래로 내려오는" 형태를 요청했고, 구현 복잡도(트랜지션, 바깥 클릭 감지 영역 계산)도 드롭다운보다 높아 채택하지 않음.
- **전역 상태(Context/Zustand 등) 도입**: 메뉴 열림 상태는 `Layout` 컴포넌트 내부에서만 쓰이므로 지역 `useState`로 충분, 불필요한 복잡도 추가 안 함.
