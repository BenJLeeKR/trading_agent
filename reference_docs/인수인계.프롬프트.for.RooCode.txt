현재 세션의 컨텍스트가 너무 길어져서 새로운 세션으로 작업을 이관하려고 해. 다음 작업자(새로운 Roo Code 세션)가 상황을 완벽하게 파악하고 끊김 없이 작업을 이어갈 수 있도록, 현재까지의 진행 상황을 요약한 `HANDOVER_TO_NEW_SESSION.md` 파일을 `plans/` 폴더에 작성해줘.

이 문서에는 다음 사항이 반드시 포함되어야 해:
1. **Current Status (현재 상태)**: 이번 세션에서 방금 완료한 핵심 작업과 가장 주요하게 변경된 파일 목록.
2. **Work In Progress (진행 중인 작업)**: 완벽히 끝나지 않았거나 테스트가 통과되지 않은 채로 중단된 작업, 그리고 발생 중인 에러 메시지나 이슈.
3. **Implicit Context (숨은 맥락 및 의사결정)**: 공식 설계 문서나 `BACKLOG.md`에는 아직 적혀있지 않지만, 코드를 짜면서 합의한 중요한 규칙, 임시 방편(Workaround), 또는 주의사항.

문서 작성을 완료하면, 세션을 종료해도 좋다고 알려줘.
