# 증권시스템 컬럼 명명규칙 v0.2 — 컬럼 메타 Rule

> 본 문서는 `securities_meta_system_dictionary_v0.5.2.md`의 핵심 테이블 컬럼 메타 구조와 12자리 고정 길이 표준테이블명 체계를 기준으로, 증권사 기간계/META 시스템에서 컬럼을 표준화하여 설계·등록·검증·운영하기 위한 **컬럼 메타 작성 Rule 문서**입니다.

| 항목 | 기준 |

## 02A. 컬럼 메타의 테이블명 참조 기준

`securities_meta_system_dictionary_v0.5.2.md`에서는 테이블명 기준을 12자리 고정 길이 표준테이블명으로 통일한다. 따라서 컬럼 메타 역시 기존 `TB_*`, `MT_*` 물리명을 직접 기준으로 삼지 않고 `STD_TBL_NM`을 공식 참조키로 사용한다.

| 표기 항목 | 의미 | 적용 기준 |
| --- | --- | --- |
| `STD_TBL_NM` | 12자리 표준테이블명 | 컬럼 메타의 공식 참조 테이블명 |
| `LOGIC_TBL_NM` | 논리테이블명 | 업무 설명 및 화면 표시용 |
| `PHYS_TBL_NM` | 실제 DB 물리명 | 운영 DB에서 실제 사용되는 테이블명 |
| `LEGACY_TBL_NM` | 기존/레거시 테이블명 | 전환, 이행, 영향도 분석용 |

### 02A.1 컬럼 메타 표기 원칙

| 원칙 | 설명 |
| --- | --- |
| 공식 참조 기준 | 컬럼 메타의 테이블 참조는 `STD_TBL_NM`을 우선한다. |
| 기존명 표기 | 기존 `TB_CUST_MST`, `TB_ORD_MST`, `MT_COLUMN` 등은 `PHYS_TBL_NM` 또는 `LEGACY_TBL_NM`으로만 표기한다. |
| 절 제목 기준 | 컬럼 메타 절 제목은 `논리테이블명 STD_TBL_NM: <표준명> / Legacy: <기존명>` 형식을 권장한다. |
| 영향도 분석 | 컬럼, 인터페이스 필드, 배치 입출력, 품질 Rule은 `STD_TBL_NM + PHYS_COL_NM` 기준으로 연결한다. |
| DDL 생성 | 신규 차세대 DB는 `STD_TBL_NM`, 기존 DB 반영은 `PHYS_TBL_NM`을 사용할 수 있다. 단 META 관계는 `STD_TBL_NM`을 기준으로 유지한다. |

### 02A.2 예시

| 논리테이블명 | STD_TBL_NM | Legacy/물리명 | 컬럼 예시 |
| --- | --- | --- | --- |
| 고객기본 | `CCUMSTBSM001` | `TB_CUST_MST` | `CUST_NO`, `CUST_NM`, `CUST_STS_CD` |
| 주문원장 | `CORORDBST001` | `TB_ORD_MST` | `ORD_DT`, `ORD_NO`, `ACNO`, `ORD_STS_CD` |
| 체결내역 | `CEXEXCDTT001` | `TB_EXEC_DTL` | `EXEC_DT`, `EXEC_NO`, `EXEC_QTY`, `EXEC_UPRC` |
| 메타컬럼 | `MMTCOLBSM001` | `MT_COLUMN` | `COLUMN_ID`, `STD_TBL_NM`, `PHYS_COL_NM`, `DOMAIN_ID` |


## 03. 컬럼 논리명 규칙

### 03.1 기본 규칙

| 항목 | 규칙 | 예시 |
| --- | --- | --- |
| 언어 | 한글 사용 원칙 | 고객번호, 주문일자 |
| 구성 | 수식어 + 핵심명사 + 성격어 | 주문 + 접수 + 일시 |
| 의미 | 업무 의미가 드러나야 함 | `처리구분`보다 `결제처리구분` 권장 |
| 중복 | 동일 의미 컬럼은 동일 논리명 사용 | 계좌번호, 고객번호 |
| 금지 | 개발자 임의 명칭, 화면명 기반 컬럼명 금지 | `값1`, `구분2`, `화면표시값` 금지 |
| 약어 | 한글 논리명에는 불필요한 약어 사용 금지 | `실명확인여부` 권장, `실명확인YN` 지양 |

### 03.2 논리명 구성 패턴

| 패턴 | 설명 | 예시 |
| --- | --- | --- |
| 주체 + 번호 | 식별자 | 고객번호, 계좌번호, 주문번호 |
| 업무 + 일자 | 업무 기준일 | 주문일자, 체결일자, 결제일자 |
| 업무 + 일시 | 이벤트 발생 시각 | 접수일시, 체결일시, 처리일시 |
| 대상 + 코드 | 코드형 값 | 고객구분코드, 계좌상태코드 |
| 대상 + 여부 | Y/N 값 | 사용여부, 삭제여부, 거래정지여부 |
| 대상 + 금액 | 금액형 값 | 주문금액, 결제금액, 수수료금액 |
| 대상 + 수량 | 수량형 값 | 주문수량, 체결수량, 보유수량 |
| 대상 + 단가 | 가격형 값 | 주문단가, 체결단가, 매입평균단가 |
| 대상 + 비율/율 | 비율형 값 | 담보비율, 이자율, 수수료율 |
| 대상 + 사유 | 설명성 값 | 거부사유, 보정사유 |
| 대상 + 메시지 | 오류/응답 메시지 | 거부메시지, 오류메시지 |

## 07. PK/FK/UK/Index 컬럼 규칙

### 07.1 PK 컬럼 규칙

| 규칙 | 설명 |
| --- | --- |
| PK 컬럼은 `NULL` 허용 불가 |
| PK 컬럼은 업무 식별 기준을 반영해야 함 |
| 거래 원장은 업무일자 + 업무번호 조합을 우선 검토 |
| 잔고/스냅샷 테이블은 기준일자 + 계좌번호 + 상품코드 + 잔고유형 조합을 우선 검토 |
| 이력 테이블은 원천 PK + 이력순번 또는 변경일시를 포함 |
| PK에 의미 없는 단일 surrogate key만 사용하는 것은 지양. 필요 시 업무키와 병행 관리 |

### 07.2 권장 PK 패턴

| 테이블 유형 | 권장 PK 패턴 | 예시 |
| --- | --- | --- |
| 고객/계좌 마스터 | 식별번호 | `CUST_NO`, `ACNO` |
| 상품 마스터 | 상품코드 | `ISU_CD` |
| 주문 원장 | 업무일자 + 주문번호 | `ORD_DT + ORD_NO` |
| 체결 원장 | 업무일자 + 체결번호 | `EXEC_DT + EXEC_NO` |
| 결제 예정 | 결제일자 + 계좌번호 + 종목코드 + 순번 | `STL_DT + ACNO + ISU_CD + STL_SEQ` |
| 종목잔고 | 기준일자 + 계좌번호 + 종목코드 + 잔고유형 + 신용구분 + 대출일자 | `BAS_DT + ACNO + ISU_CD + BAL_TP_CD + CRD_DVSN_CD + LOAN_DT` |
| 예수금잔고 | 기준일자 + 계좌번호 + 통화코드 | `BAS_DT + ACNO + CURR_CD` |
| 이력 | 원천 PK + 이력순번 | `ORD_DT + ORD_NO + HIST_SEQ` |
| 코드상세 | 코드그룹 + 코드 | `GRP_CD + CD` |

### 07.3 FK 컬럼 규칙

| 규칙 | 설명 |
| --- | --- |
| FK 컬럼명은 참조 대상 PK 컬럼명을 그대로 사용한다. |
| 물리 FK 제약을 걸지 않더라도 META상 논리 FK는 반드시 관리한다. |
| 인터페이스/스테이징 테이블은 물리 FK 생략 가능하나, 매핑 메타는 관리해야 한다. |
| 코드 컬럼은 공통코드 테이블에 대한 논리 FK로 관리한다. |
| FK 참조 무결성 품질검증 SQL을 별도 Rule로 등록한다. |

## 08. NULL 허용 및 기본값 규칙

### 08.1 NULL 허용 기준

| 컬럼 유형 | NULL 허용 기준 |
| --- | --- |
| PK 컬럼 | 절대 불가 |
| FK 핵심 업무 컬럼 | 원칙적으로 불가. 단, 업무상 미확정 상태가 존재하면 허용 |
| 코드 컬럼 | 상태/구분/유형의 핵심 값은 불가 |
| 금액/수량 컬럼 | 계산 대상이면 원칙적으로 `0` 기본값 사용 |
| 일시 컬럼 | 이벤트가 발생하지 않은 상태가 업무적으로 의미 있으면 허용 |
| 종료일자 | 미종료 상태를 표현하기 위해 NULL 또는 `99991231` 중 회사 표준을 선택 |
| 메시지/설명 | 선택 입력이면 허용 |
| Audit 컬럼 | 등록/변경 기본 Audit은 운영 테이블에서 불가 권장 |

### 08.2 기본값 기준

| 컬럼 | 기본값 | 설명 |
| --- | --- | --- |
| `USE_YN` | `Y` | 신규 등록 기본 사용 |
| `DEL_YN` | `N` | 논리삭제 기본 미삭제 |
| `DATA_VER_NO` | `1` | 최초 데이터 버전 |
| `PROC_STS_CD` | `00` 또는 `READY` | 처리 전 상태 |
| `ERR_CNT` | `0` | 오류 건수 |
| `*_AMT` | `0` | 금액 계산 컬럼 |
| `*_QTY` | `0` | 수량 계산 컬럼 |
| `*_YN` | 업무별 기본값 | 무조건 `N`으로 고정하지 않고 업무 의미 검토 |

## 10. 금액·수량·가격·율 컬럼 규칙

### 10.1 금액 컬럼

| 규칙 | 설명 |
| --- | --- |
| 금액 컬럼은 `_AMT` suffix를 사용한다. |
| 도메인은 `AMT`를 사용한다. |

| 매수/매도 방향에 따라 부호를 사용할지, 별도 매매구분코드를 사용할지 업무 표준을 명확히 한다. |

| 예시 | 설명 |
| --- | --- |
| `ORD_AMT` | 주문금액 |

| `FEE_AMT` | 수수료금액 |
| `TAX_AMT` | 세금금액 |
| `EVAL_PL_AMT` | 평가손익금액 |

### 10.2 수량 컬럼

| 규칙 | 설명 |
| --- | --- |
| 수량 컬럼은 `_QTY` suffix를 사용한다. |
| 도메인은 `QTY`를 사용한다. |
| 해외주식, 채권, 펀드, 소수점거래를 고려하여 `NUMBER(18,6)` 권장 |
| 일반적으로 음수 수량은 금지하고, 매매 방향은 별도 코드로 표현한다. |
| 보정/취소 수량은 별도 컬럼 또는 음수 허용 여부를 업무적으로 명시한다. |

### 10.3 가격/단가 컬럼

| 규칙 | 설명 |
| --- | --- |
| 단가는 `_UPRC`, 가격은 `_PRC`를 사용한다. |
| 도메인은 `PRICE`를 사용한다. |
| 호가단위 검증이 필요한 경우 `TICK_SIZE_CD` 또는 시세/상품 메타와 연계한다. |
| 채권 가격은 액면 기준 가격, 만원당 단가 등 기준을 컬럼 설명에 반드시 명시한다. |

### 10.4 비율/율 컬럼

| 규칙 | 설명 |
| --- | --- |
| 비율/율 컬럼은 `_RT` suffix를 사용한다. |
| 도메인은 `RATE`를 사용한다. |
| 값 저장 기준이 `%`인지 소수인지 명시해야 한다. |
| 예: 5%를 `5.00000000`으로 저장할지 `0.05000000`으로 저장할지 도메인 설명에 명시한다. |

| 일자 컬럼은 `_DT` suffix를 사용한다. |
| 기준일자는 `BAS_DT`, 거래일자는 `TR_DT`, 주문일자는 `ORD_DT`처럼 업무 접두어를 붙인다. |
| `CHAR(8)`을 사용할 경우 `YYYYMMDD` 검증을 필수 적용한다. |
| 실제 달력일 검증, 영업일 검증, 휴장일 검증 여부를 별도 품질 Rule로 관리한다. |

### 11.2 일시 컬럼

| 규칙 | 설명 |
| --- | --- |
| 일시 컬럼은 `_DTTM` suffix를 사용한다. |
| `TIMESTAMP(6)` 사용 권장 |
| 거래소/FEP/채널계 이벤트 시각은 원천시각과 내부접수시각을 분리할 수 있다. |
| 타임존이 필요한 해외업무는 `TZ_CD` 또는 UTC 변환 기준을 함께 관리한다. |

SELECT table_name, column_name

## 부록 C. 컬럼 메타 Rule ID 체계

컬럼 메타 Rule은 다음 형식을 사용한다.

```text
COL-{영역}-{순번}
```

| 영역 | 의미 |
| --- | --- |
| `NM` | 명명 규칙 |
| `TYPE` | 데이터 타입/도메인 |
| `KEY` | PK/FK/UK |
| `NULL` | NULL/기본값 |
| `CODE` | 코드 컬럼 |
| `AUD` | Audit 컬럼 |
| `SEC` | 개인정보/보안 |
| `IF` | 인터페이스 추적 |
| `BAT` | 배치 추적 |
| `DQ` | 품질 검증 |

예시:

| Rule ID | 설명 |
| --- | --- |
| `COL-NM-001` | 물리컬럼명 대문자/숫자/underscore 검증 |
| `COL-TYPE-001` | suffix와 도메인 일치 검증 |
| `COL-CODE-001` | 코드그룹 필수 검증 |
| `COL-AUD-001` | 운영 테이블 Audit 컬럼 필수 검증 |
| `COL-SEC-001` | 개인정보 컬럼 보안속성 필수 검증 |
| `COL-DQ-001` | 업무 품질검증 Rule 연결 여부 검증 |
